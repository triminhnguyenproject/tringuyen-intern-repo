# Roles in an Agile Team

## Research Question

**What are the key roles in an Agile team, and what does each role do?**

- Developers: Build the product
- QA (Quality Assurance): Ensure the product work correctly and meet quality standart
- Product Manager (PM): Define the product strategy, ensure the team builds product meet requirement and features
- Agile Project Manager (Agile PM): Plan and track project progress, facilitate Agile ceremonies
- Other supporting roles

**How do these roles collaborate in Kanban vs. Scrum?**

- Kanban: PM continuously reprioritizes backlog. Developers working on completing features on the board, adding new item if needed. QA would tests each delivered item as they being add and move through the board. Agile PM would monitor the amount of work in progress. No sprint ceremonies is required.
- Scrum: The team will need a sprint planning for each sprint. PM present required items, developers need to estimate the work and the team select tasks for that sprint. Throughout the sprint, developers build the product and QA conduct testing during the sprint. Agile PM run planning and prepare stand-ups. At the end of the sprint, the team demonstrate completed work and getting feedback from stakeholders; the team would reflect, plan to improve the process and product for the next sprint.
