# Agile Workflows & Kanban

## Research Question

**How does a Kanban board work, and how does it help manage workflow?**
Kanban use a visualization tool called Kanban boards to manage workload and flow. On the board, work is displayed in columns, typically represent a stage of work. The most basic board have columns such as "To Do", "In Progress" and "Done". Each task / item will be show on the board as a card. When being create, start or complete, it will move forward in the board to the next column until the last one (usually the "Done").

Using a Kanban board, teams could easily visualize the current workflow. The board would clearly show how many task are currently in the plan, the team could see if there are too much task in progress and limit it. 

**What do the different columns on a Kanban board represent?**
Each columns usually represent a stage of work, such as "To Do", "In Progress" and "Done".

**How do tasks move through the board, and who is responsible for updating them?**
When a task is create, it would be put on a card and add to the column "To Do". When someone start working on the task, they move it into the next (In Progress) column. After the task is satisfy and get the result, it will be moved to "Done" column. Before getting to the last column (Done), the Kanban task card could move through some columns like "Testing", "Reviewing" if needed.

The person doing the task is responsible for updating the task. For instance:

- A developer start a task, they move it to "In Progress"
- When the work is done, move it to "Testing"
- When testing is complete, tester mover it to "Done"

**What are the benefits of limiting work in progress (WIP)?**
Help team focusing on completing singular task, reduce multitasking, which improve productivity. It also help identify bottleneck, if any column fill up and reach limit, the team could know which step of the workflow is slowing down the process. Limiting WIP prevent team from being overwhelmed by to much tasks, reduce overload and stress.
